# Style Guide
## A Guide for new contributors

### String Replacing
Do not replace inline string replacement with concatenating dissimilar variable formats. Ex. Strings and Integers.

### Variables
I have specifically organized by source code into constants, globals, and local variables. I do not use instance variables because I don't use object-oriented programming, this is procedural wuth pre-fab functio

### Quotes Specifically
Do not replace double quotes with single quotes and reformatt comments, these repo is designed as an introduction for Ruby programmers, and not intended as software for so called "seasoned developers".

### Breaking Changes
Do not continuously issue breaking changes that undermine the integrity of this procedurally programmed format, without consulting the rest of the team, which for right now is mostly myself. I can't afford to pay new people at the moment. And I don't do free labour.
